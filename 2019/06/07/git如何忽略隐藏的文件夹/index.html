往GitHub上推代码的时候不小心把项目的历史版本记录都推上去了.于是写了个gitignore忽略掉.

但一开始怎么改变gitignore的规则都无法删除GitHub上的.history文件夹.

最终发现了一个”笨方法”,还算管用:

  1. 真的删除该文件夹(改名字也行,但要把改名后的文件夹写到.gitignore中去.语法就像上面.history/*那样)

  2. 把这个版本推上去

  3. 恢复刚才删除的文件夹

  4. 修改.gitignore文件,把一开始要忽略的隐藏文件夹加上
